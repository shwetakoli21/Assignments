Generics in Java — Definition & Introduction

Introduction

Introduced in Java 5, generics enable parameterized types so a single class or method can work with different data types. For example, the Java Collections framework uses generics to ensure that containers (like ArrayList and HashMap) store only the intended type of elements. Generics achieve two main goals:

Definition

Generics in Java are a language feature that allows classes, interfaces, and methods to be parameterized by types. Using generics, programmers can write code that operates on objects of various types while providing compile-time type safety and eliminating the need for explicit type casts.

  1. Type Safety: Errors such as invalid type assignments are caught at compile time rather than at runtime.
  2. Reusability: The same code can be reused for different data types without duplication.

Wrapper Classes

1. Introduction

Wrapper classes in Java are used to represent primitive data types as objects. They allow primitive values to be used in places where only objects are allowed, such as Collections and Generics.

2.Definition

Wrapper classes are predefined classes in Java that wrap (enclose) a primitive value inside an object. Each primitive type has a corresponding wrapper class.

3. Purpose of Wrapper Classes

  • Convert primitive data types into objects.
  • Enable use of primitives in Collections (like ArrayList, HashMap).
  • Support autoboxing and unboxing.
  • Provide utility methods such as parsing and conversion.
  • Allow null values, which primitives cannot represent.

Collections in Java

Definition

Collection is set of object stored in single variable.

A Collection in Java is a framework that provides classes and interfaces to store, manage, and manipulate groups of objects efficiently. A collection is a group of objects (called elements) stored together in a single unit.

Why Collections?

  • Arrays have fixed size and limited functionality.
  • Cannot easily perform searching, sorting, or data manipulation.
  • Collections framework provides dynamic and flexible data structures.

Main Interfaces

1.List

  • Stores elements in order.
  • Allows duplicate elements.
  • Examples: ArrayList, LinkedList, Vector

2.Set

  • Does not allow duplicate elements.
  • No guaranteed order.
  • Examples: HashSet, LinkedHashSet, TreeSet

3.Map

  • Stores data in key–value pairs.
  • Keys must be unique.
  • Examples: HashMap, LinkedHashMap, TreeMap

Advantages of Collections

  • Dynamic size
  • Built-in methods for searching, sorting, and iteration
  • Easy data manipulation
  • Memory efficient
  • Supports different data structures
