Difference Between Primary Key and Foreign Key

Primary Key

A Primary Key is a column or a group of columns that uniquely identifies each record in a table.

  • Uniquely identifies each row
  • Cannot contain NULL values
  • Duplicate values are not allowed
  • Only one Primary Key per table
  • Ensures entity integrity

Example

CREATE TABLE Student (
    student_id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

Foreign Key

A Foreign Key is a column that is used to create a link between two tables. It refers to the Primary Key of another table.

  • Can contain duplicate values
  • Can contain NULL values
  • More than one Foreign Key is allowed in a table
  • Ensures referential integrity
  • Used to maintain relationship between tables

Example

CREATE TABLE Marks (
    marks_id INT,
    student_id INT,
    FOREIGN KEY (student_id) REFERENCES Student(student_id)
);
