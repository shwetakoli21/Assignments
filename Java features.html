Java 8 Features

Java 8, released in March 2014, introduced several major features that modernized Java programming.

1. Lambda Expressions

Lambda expressions allow writing anonymous functions and enable functional programming in Java.


List list = Arrays.asList("A", "B", "C");
list.forEach(item -> System.out.println(item));

2. Functional Interfaces

An interface with only one abstract method. Can be used with lambda expressions.


@FunctionalInterface
interface MyInterface {
    void display();
}

3. Stream API

Allows processing collections in a functional style with operations like filter, map, reduce, and collect.


List numbers = Arrays.asList(1,2,3,4,5);
numbers.stream()
       .filter(n -> n % 2 == 0)
       .forEach(System.out::println); // prints 2, 4

4. Default Methods in Interfaces

Interfaces can have default implementations of methods without affecting existing implementations.


interface MyInterface {
    default void greet() {
        System.out.println("Hello from default method");
    }
}

5. Method References

Short-hand for lambda expressions calling existing methods.


List list = Arrays.asList("A", "B");
list.forEach(System.out::println);

6. Optional Class

Helps avoid NullPointerException by encapsulating optional values.


Optional name = Optional.ofNullable(null);
System.out.println(name.orElse("Default Name")); // prints "Default Name"

7. New Date and Time API

Modern API for dates and times (java.time package).


LocalDate date = LocalDate.now();
LocalTime time = LocalTime.now();
LocalDateTime dateTime = LocalDateTime.now();
System.out.println(dateTime);

8. Nashorn JavaScript Engine

Allows executing JavaScript code within Java applications.


ScriptEngineManager manager = new ScriptEngineManager();
ScriptEngine engine = manager.getEngineByName("nashorn");
engine.eval("print('Hello JavaScript in Java 8');");

9. Parallel Array Sorting

Arrays can be sorted using multiple threads with Arrays.parallelSort().


int[] arr = {5, 3, 2, 8};
Arrays.parallelSort(arr);
System.out.println(Arrays.toString(arr)); // [2,3,5,8]

10. New Collection Methods

Additional methods added to collections for convenience.

  • forEach() – iterate elements
  • removeIf() – remove elements based on a condition
  • replaceAll() – replace elements
  • sort() – sort elements

List list = new ArrayList<>(Arrays.asList("B","A","C"));
list.sort(String::compareTo);
list.forEach(System.out::println);

11. Repeating Annotations

Allows applying the same annotation multiple times on a class or method.


@MyAnnotation(value="One")
@MyAnnotation(value="Two")
class MyClass {}

12. Type Annotations

Annotations can be applied to types for better type checking.


List<@NonNull String> names;

in short

  • Lambda Expressions enable functional programming.
  • Streams provide a powerful way to process collections.
  • Optional helps avoid null pointer errors.
  • Default methods add new functionality to interfaces.
  • New Date/Time API replaces old java.util.Date.
  • Method references provide concise syntax.
  • Nashorn engine allows JS execution in Java.
  • Collections have new useful methods for better data manipulation.
  • Annotations improvements: repeating and type annotations.
