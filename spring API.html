1. REST API

-API = Application Programming Interface
-A REST API is a way for different applications to talk to each other over the internet using standard rules.
-Means technique that exchange data with backend server.
-REST uses standard HTTP methods like GET, POST, PUT, DELETE.

2. GET & POST

GET- Used to fetch data from the server.
POST- Used to send data to the server (like inserting a new record).

3. JSON

A lightweight format to send and receive data. Example format: 

{
  "id": 105,
  "name": "shweta koli"
}
JSON uses key-value pair format
Stands for JavaScript Object Notation

4. Controller

A Controller is a class that handles external HTTP requests.
Most APIs are written inside controllers.
Use @RestController
The controller receives requests from the user (client) and decides what to do next—like calling the service and returning the response.
@RestController

public class StudentController {

    @GetMapping("/students")
    public String getStudents() {
        return "All Students";
    }
}

It maps URLs to methods using annotations like:
•	@GetMapping
•	@PostMapping
•	@PutMapping
•	@DeleteMapping

5. CRUD

-(Create,read,update,delete) This are basic 4 operations

•  Create → POST
•  Read → GET
•  Update → PUT
•  Delete → DELETE

6. Service

Service contains business logic (main processing). Controller sends the request here.
Controller → Service → Repository
• Service never talks directly to the user.
• It only talks to the Controller and Repository.

@Service
public class StudentService {
   public List getAllStudents() {
       return repo.findAll();
   }
}

7. Repository

Manage db op like save update,delete and find.
Repository is part of Data Access Layer (DAO).
Use JpaRepository

 @Repository
public interface StudentRepo extends JpaRepository { }

8. Entity / Model

This is a class that represents a database table (fields = columns).
Class which have variables
Every Entity must have @Id
@Entity makes the class ready for ORM (Hibernate).

 @Entity
public class Student {
@Id
  private int id;
  private String name;
}

9. PathVariable

Used to pass values inside the URL.
-GET /student/5
Works with URL parameters.
Extracts value from URL.
Controller: 

@GetMapping("/student/{id}")
public Student getStudent(@PathVariable int id) {
    return service.getStudent(id);
}

10. RequestBody

- Used to get JSON format data from the client
Used only in POST / PUT requests usually.

@PostMapping("/students")
public Student addStudent(@RequestBody Student s) {
return service.addStudent(s);
}

11. Postman Testing

Postman is a software tool used to test APIs without creating a frontend.

14. Flow

•  Client sends request (GET/POST/PUT/DELETE)
•  Controller receives it
•  Controller calls Service
•  Service calls Repository
•  Repository interacts with Database
•  Response returns back to client
Step What happened POST /student Send data Controller Accept request Service Processing according to rules Repository Save in db Response Then “Student Saved Successfully”

In Spring Boot, we never manually create objects using new.
Spring creates them automatically using:

•	@Autowired

•	@Component
Example:
